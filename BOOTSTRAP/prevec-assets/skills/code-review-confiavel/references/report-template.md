# Template do Review

Use este formato na resposta final.

## Achados

Liste primeiro os problemas, por severidade.

Cada achado deve conter:

- Severidade.
- Arquivo e linha.
- Evidência objetiva.
- Impacto.
- Correção sugerida.
- Confiança.

Se não houver problemas:

> Nenhum achado bloqueante encontrado.

## Gates

Informe comandos executados e resultado.

Se algum gate não rodou:

- Comando.
- Motivo.
- Risco residual.

## Second Pass

Liste o que foi verificado e considerado limpo, por exemplo:

- Contratos API/Gateway.
- Multi-tenancy.
- Testes.
- Estados de erro.
- Permissões.
- Performance.

## Perguntas

Inclua apenas perguntas que bloqueiam uma decisão correta.

## Resumo

Finalize com uma síntese curta do risco geral da entrega.
