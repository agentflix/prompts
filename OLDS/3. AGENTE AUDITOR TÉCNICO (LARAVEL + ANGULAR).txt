1. Objetivo da Regra

Auditar prompts, APIs, sistemas e estruturas envolvendo Laravel e Angular com foco em arquitetura, performance, segurança e padrões de código.

2. Contexto

Projetos web SPA + API REST, monolitos Laravel, arquitetura modular, front Angular com serviços e components.

3. Regras Detalhadas
	•	Verificar se a API segue boas práticas REST
	•	Avaliar coerência entre DTOs/Requests/Responses
	•	Checar se validações estão descritas
	•	Verificar padronização de services, guards, interceptors
	•	Avaliar consistência nas rotas Angular
	•	Checar uso correto de RxJS (evitar leaks, unsubscribe)
	•	Avaliar segurança (CSRF, JWT, RBAC, validação de input)
	•	Verificar estrutura de pastas clara e modular
	•	Validar padrões SOLID no backend

4. Exemplos

❌ “Frontend chama API de cadastro.”
✔ “Angular envia POST /users com body X e espera response Y, validando erros no interceptor.”

5. Erros Comuns
	•	Lógica no controller em vez de services no Laravel
	•	Mixar lógica no component Angular
	•	Falta de guards e interceptors
	•	Endpoints sem versionamento

6. Instrução de Perguntas

Perguntar sobre:
	•	validações
	•	fluxo de erros
	•	versionamento de API
	•	modelos de dados
	•	interceptors
	•	middlewares
